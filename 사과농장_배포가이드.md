# 사과농장 배포 가이드

---

## 구조 한눈에 보기

```
[개발 PC]                        [NAS web/update/]           [사용자 PC]
launcher.py 수정                                             launcher.exe 실행
      ↓                                                            ↓
build_launcher.py 실행           licenses.json  ←───────── 라이선스 체크
      ↓                          version.json   ←───────── 버전 체크
launcher.exe 생성                update.zip     ←───────── 업데이트 다운로드
      ↓                                                            ↓
dist\ent_bot_deploy\ 배포                                  app\ 압축해제 후 실행
```

---

## 시나리오 1 — 봇 코드만 수정했을 때 (가장 흔한 경우)

> ent_bot_engine_template.py, auto_login.py 등 앱 파일만 고쳤을 때

1. `build_update.bat` 더블클릭 (또는 `python build_update.py`)
2. 버전 입력 (예: `1.0.1`) → Enter
3. 생성된 두 파일을 NAS `web/update/` 폴더에 업로드 (덮어쓰기)
   - `update.zip`
   - `version.json`
4. 끝 — 사용자가 런처 실행하면 자동 업데이트됨

---

## 시나리오 2 — 런처 자체(launcher.py)를 수정했을 때

> 런처 UI, 라이선스 로직, 업데이트 로직 등을 고쳤을 때

1. 런처 창이 열려 있으면 **반드시 닫기**
2. `python build_launcher.py` 실행 (또는 `build_launcher.bat`)
3. `dist\ent_bot_deploy\launcher.exe` 생성됨
4. `dist\ent_bot_deploy\` 폴더 전체를 사용자 PC에 다시 배포

> 런처는 자동 업데이트 안 됨 — 직접 교체 필요

---

## 시나리오 3 — 새 PC 등록 (라이선스 추가)

1. 사용자에게 런처 실행 후 PC ID(12자리 hex) 받기
   - 런처 상단 `PC ID:` 옆에 표시됨, 복사 버튼 있음
2. NAS에서 `web/update/licenses.json` 열기
3. 항목 추가:
   ```json
   "PC아이디": "YYYY-MM-DD"
   ```
   예시:
   ```json
   {
       "c4e92f29b60a": "2026-12-31",
       "aabbccddeeff": "2026-12-31"
   }
   ```
4. 저장 후 NAS 업로드 (덮어쓰기)
5. 사용자가 런처에서 "업데이트 확인" 버튼 누르면 즉시 적용

---

## 시나리오 4 — 새 PC에 처음 배포

1. `dist\ent_bot_deploy\` 폴더를 새 PC에 복사
   ```
   launcher.exe
   launcher_config.json
   install_deps.bat
   ```
2. 새 PC에서 `install_deps.bat` 먼저 실행 (Python 패키지 설치)
3. `launcher.exe` 실행
4. PC ID 확인 후 licenses.json에 등록 (시나리오 3)
5. 런처에서 "업데이트 확인" → update.zip 다운로드 → 앱 설치됨

---

## NAS 파일 경로 정리

| 파일 | 역할 |
|------|------|
| `web/update/licenses.json` | PC별 라이선스 만료일 |
| `web/update/version.json` | 현재 배포 버전 번호 |
| `web/update/update.zip` | 앱 파일 전체 압축 |

접속 URL: `http://dh2302.synology.me/update/`
